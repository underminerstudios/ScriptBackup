Multiple DAE exporter
author: Marc Marti - marc.marti@gameloft.com
DESCRIPTION:
  • This script will allow you to export all the selected objects as a separated DAE files inside the selected folder.
  • Quite usefull for massive environment object exportations.
  • IMPORTANT: The objects will be exported with the same name as the are named inside MAX, so, it's imperative to label and name properly all the objects in the scene.
  • IMPORTANT: All the objects should have materials, unless the NO MATERIAL option is enabled.
OPTIONS:
  • Exportation path: The folder where all the objects will be exported.
  • Set as default: Records the path for future exportations (the memo file is saved in the c:/svn folder, so you need this folder to enable this option.).
  • Prefix: String that will be setted in front of the object name automatically during the exportation.
  • Sufix: String that will be setted at the end of the object name automatically during the exportation.
ADVANCED OPTIONS:
  • Export Excel prepared LOG: This options saves a report of data of the scene specially formated to be pasted inside an excel file.
  • Export hierarchy as single DAE: Exports all the linked objects as a sigle DAE file.
  • Export NON-MESH objects: The script will be able to export dummies lights and other non-mesh elements of the scene.
  • Export objects with special materials: The script will be able to export collision materials and other clara-specailly formated mats.
  • Export objects with NO-MATERIALS: The objects without material will be exported too.
INSTRUCTIONS:
  • Select the objects that you want to export.
  • Click the [Export Objects] button.
KNOWN ISSUES:
  • To avoid material problems in clara is highly recommended transform all the materials to [standard] format before exportation (Vray materials and normalBump slots are not functional in clara)
