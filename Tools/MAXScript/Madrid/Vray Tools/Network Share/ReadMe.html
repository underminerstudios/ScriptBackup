Network Share
author: Norman Schaar (norman.schaar@gameloft.com)
  • In order to be able to render your textures need to be accessible through the network first. This tool will allow you to do just that with a single click
  • This tool will list all the files referenced by the 3dsMax scene. If the color is green it means the file has been found. If a file is red, it means it's not found.
  • To ensure that the rendering works properly all files should be green. The script will still work if some textures are not found, but the result of the DR will not be fine.
  • If something looks off in the render, chances are (at least 90% of the time) that one of the missing textures is causing the problem.
  • If you change paths make sure to launch the tool again in order to refresh the list.
  • Once all your paths are green you can click "Share".
  • This will essentially, look for all the different folders used, and will share those folders in the network. It will also change the paths to the network paths, so that the nodes can access it.
  • WARNING ARTISTS:
  • Don't put textures in your Desktop folder, in drive root folders, system32 folder o any other crazy place you can think of... -_-' The script won't be able to share those folders!
  • Make sure to click "UnShare" after you are done rendering.
